Detailed Description

Handles collisions of the hand component.

  • Created on: October 2013
  • Author: Frederik Beuth
  • Mechanism in the case an object is usable
    • The tag "usable" denotes that the object can either be grasped, pointed or interacted with.
    • The object itself has to be tagged with "usable"
    • agentHand and object needs both collider and rigidbodies.
    • istrigger and isKenematic have to be off (see next case), so we use the OnCollisonEnter function.
    • OnCollisionEnter recognizes that the object should be used if: 1) the object is tagged with usable, 2) the hand is tagged with agentHand, 3) the action is executed dependent if an msgGrasp, msgPoint or msgInteraction is currently executed.
    • For detailed procedure how to grasp, see the comments in the source code.
  • Mechanism in the case an object is NOT usable
    • Object is not usable, if no tag "usable" is applied.
    • Intended behaviour is that the hand pushes the object away.
    • So agentHand and object needs both collider and rigidbodies.
    • istrigger and isKenematic have to be off for both to use the physics calculation.
    • Through physics, the object is pushed away.
    • The hand would also be pushed away. To avoid this, position and rotation have to be frozen.

Settings for the hand of an agent:

  • to be tagged as "hand"
  • collider with isTrigger=off
  • collider with freezed position and rotation
  • rigidbody with useGravity=off
  • this script

Settings for all graspable objects

  • to be tagged as "agentHand"
  • collider
  • rigidbody
